# llm-workbench-registry

Curated registry of **modes** and **model families** for
[LLM Workbench](https://github.com/Dominga/LLM-Worckbench).

The Workbench app fetches `index.json` from this repository (or any fork
serving the same schema) and lets users install individual artifacts —
modes (system prompt + tool whitelist) and families (model-family
metadata used for UI grouping and prompt variants).

## Layout

```
modes/
  <id>/
    <id>.toml          # mode definition (see docs)
    <id>.system.md     # default prompt template
    <id>.qwen3.system.md   # optional family-specific variant

families/
  <id>/
    <id>.toml          # family metadata

index.json             # generated by CI; never edit by hand
```

`index.json` is rebuilt by the GitHub Action in
`.github/workflows/build-index.yml` on every push to `main`. The
Workbench app reads the file at:

```
https://raw.githubusercontent.com/Dominga/llm-workbench-registry/main/index.json
```

## Contributing

1. Fork the repo.
2. Add a new entry under `modes/<id>/` or `families/<id>/`.
3. Bump the `version` field in your `<id>.toml` whenever you ship a
   meaningful change — the app uses it to drive update notifications.
4. Open a pull request. The CI rebuilds `index.json` after merge.

See [docs/REGISTRY_FORMAT.md](docs/REGISTRY_FORMAT.md) for the full
schema reference and authoring tips.
